# 小红制作蛋糕

## 题目描述

- n种蛋糕，每种最多制作一次
- 门槛 x[i]：当前甜度 >= x[i] 才能制作
- 消耗体力 z[i]，获得甜度 y[i]
- 初始甜度 w，体力 e，最多制作 k 个
- 求最大甜度，以及达到该甜度所需的最少蛋糕数

## 解法：带门槛的 01 背包

### 1. 预处理：按门槛排序

因为蛋糕有门槛限制，所以先制作门槛低的，再制作门槛高的更合理。

```python
# 原始数据
x = [0, 5, 5, 8, 9]    # 门槛
y = [5, 3, 4, 10, 6]   # 甜度
z = [10, 20, 30, 40, 50]  # 体力

# 打包排序后
cakes = [(0, 5, 10), (5, 3, 20), (5, 4, 30), (8, 10, 40), (9, 6, 50)]
#           门槛   甜度   体力
```

### 2. 动态规划（DP）是什么？

**通俗理解**：DP 就是"表格法"，把大问题拆成小问题，逐步填表。

我们定义一个表格：
```
dp[j][t] = 制作了 j 个蛋糕，消耗了 t 体力，当前甜度是多少
```

- j：制作的蛋糕数量（0 ~ k）
- t：消耗的体力（0 ~ e）
- 值：能达到的最大甜度（-1 表示达不到）

### 3. 初始化

```python
dp = [[-1] * (e + 1) for _ in range(k + 1)]
dp[0][0] = w  # 不做蛋糕，甜度就是初始的 w
```

解释：制作 0 个蛋糕，消耗 0 体力，甜度是 w。

### 4. 状态转移（核心）

遍历每个蛋糕，决定做还是不做：

```python
for x_i, y_i, z_i in cakes:  # 遍历每个蛋糕
    # 倒序遍历（01背包必须倒序，避免同一个蛋糕被用多次）
    for j in range(k - 1, -1, -1):
        for t in range(e - z_i, -1, -1):
            # 如果这个状态可达
            if dp[j][t] != -1:
                # 且当前甜度 >= 门槛，才能制作
                if dp[j][t] >= x_i:
                    # 制作这个蛋糕后的新状态
                    new_j = j + 1
                    new_t = t + z_i
                    new_sweet = dp[j][t] + y_i
                    # 取较大值
                    dp[new_j][new_t] = max(dp[new_j][new_t], new_sweet)
```

### 5. 手动模拟（简化版）

假设 e=50, k=3，简化理解：

```
初始状态：
dp[0][0] = 0  （甜度0，体力0）

处理蛋糕 (门槛0, 甜度5, 体力10)：
  dp[1][10] = max(-1, 0+5) = 5

处理蛋糕 (门槛5, 甜度3, 体力20)：
  从 dp[1][10]=5 开始（满足门槛5）
  dp[2][30] = 5 + 3 = 8

处理蛋糕 (门槛5, 甜度4, 体力30)：
  从 dp[1][10]=5 开始
  dp[2][40] = 5 + 4 = 9
  从 dp[2][30]=8 开始
  dp[3][60] 超出了体力范围

... 以此类推
```

### 6. 完整代码

```python
n, w, e, k = map(int, input().split())
x = list(map(int, input().split()))  # 门槛
y = list(map(int, input().split()))  # 甜度
z = list(map(int, input().split()))  # 体力

# 按门槛排序
cakes = sorted(zip(x, y, z))

# DP 表：dp[蛋糕数][体力] = 最大甜度
dp = [[-1] * (e + 1) for _ in range(k + 1)]
dp[0][0] = w  # 初始状态

# 遍历每个蛋糕
for x_i, y_i, z_i in cakes:
    # 倒序遍历（01背包）
    for j in range(k - 1, -1, -1):
        for t in range(e - z_i, -1, -1):
            if dp[j][t] >= x_i:  # 满足门槛
                dp[j + 1][t + z_i] = max(dp[j + 1][t + z_i], dp[j][t] + y_i)

# 找答案：最大甜度 和 对应的最少蛋糕数
max_sweet = w
min_cakes = 0
for j in range(k + 1):
    for t in range(e + 1):
        if dp[j][t] > max_sweet:
            max_sweet = dp[j][t]
            min_cakes = j
        elif dp[j][t] == max_sweet and dp[j][t] != -1 and j < min_cakes:
            min_cakes = j

print(max_sweet, min_cakes)
```

### 7. 复杂度分析

| 操作 | 复杂度 |
|------|--------|
| 排序 | O(n log n) |
| DP | O(n × k × e) |
| 空间 | O(k × e) |

本题数据：n ≤ 100, k ≤ 100, e ≤ 80000，最坏 8×10^8 次运算，但实际因为有很多状态不可达，运行快很多。
